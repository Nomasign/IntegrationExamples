# Step 1 — Creating a NomaSign Account

> @confirm: Which plans should be used for Integration API access in this guide?

You need a NomaSign account on a plan that includes the **Integration** feature.

## Steps

1. Go to [app.nomasign.com](https://app.nomasign.com)
2. Click **Sign Up** and create your account (or log in if you already have one)
3. Subscribe to a plan that includes **Integration**

## Already have an account? @confirm

Verify your plan includes the Integration feature:

1. Log in at [app.nomasign.com](https://app.nomasign.com) as the **org owner**
2. Go to **Account Settings** → **Subscription**
3. Confirm that **Integration** is listed as an included feature

@confirm account
> **Don't see "Integration" on your plan?** Upgrade via **Account Settings** → **Subscription** → **Upgrade**, or contact [support@nomasign.com](mailto:support@nomasign.com). Only the org owner can view or change the subscription—ask your organization's account owner to verify integration access.

## Testing & sandbox

NomaSign does not have a separate sandbox environment. Integration features work in production — test freely by sending documents to your own email. API calls are not billed per-call; they're included with the Integration feature.
